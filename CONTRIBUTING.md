# JobBridge 기여 가이드

JobBridge에 관심을 가져 주셔서 감사합니다. 이 프로젝트는 장애인의 구직 정보 접근성과 직무 탐색 경험을 개선하는 오픈소스 서비스입니다. 작은 문서 수정부터 접근성 개선, 데이터 파이프라인 검증까지 모두 환영합니다.

기여하기 전에 [행동강령](CODE_OF_CONDUCT.md)과 [보안 정책](SECURITY.md)을 읽어 주세요.

## 가장 먼저 지켜야 할 원칙

- 이슈, 커밋, 테스트 픽스처, 스크린샷에 실제 이용자의 이름, 이메일, 장애 정보, 이력서, 인증 토큰을 넣지 마세요.
- 장애 관련 표현은 당사자를 존중하는 구체적이고 중립적인 언어를 사용하세요.
- 추천 결과를 취업 가능성이나 적합성의 확정적 판단처럼 표현하지 마세요.
- 데이터, 모델, 이미지, 글꼴을 추가할 때는 원출처, 버전, 라이선스, 수정 여부를 함께 기록하세요.
- 비상업·변경금지·접근승인 조건이 있는 데이터나 모델은 기본 배포물에 추가하지 마세요. 필요하다면 먼저 이슈에서 사용 가능성을 논의하세요.

## 기여 절차

1. 기존 이슈를 검색합니다.
2. 버그, 기능, 접근성 문제에 맞는 이슈 양식으로 변경 이유와 범위를 공유합니다.
3. 저장소를 포크하고 짧은 작업 브랜치를 만듭니다. 예: `fix/keyboard-focus`, `docs/model-card`.
4. 한 PR에는 하나의 문제만 담고 관련 테스트와 문서를 함께 갱신합니다.
5. PR 템플릿의 확인 항목을 완료하고 리뷰 의견에 응답합니다.

작은 오탈자 수정은 바로 PR을 열어도 됩니다. 구조, 개인정보 처리, 인증, 추천 로직, 데이터 또는 모델 라이선스에 영향을 주는 변경은 구현 전에 이슈를 열어 주세요.

## 개발 환경

Windows PowerShell 예시입니다.

```powershell
git clone https://github.com/freeboxwork/JobBridge-OSS.git
Set-Location JobBridge-OSS
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
$env:PYTHONPATH="$PWD\Services\jobbridge_inference"
python -m jobbridge_inference.http_server --host 127.0.0.1 --port 8787
```

민감한 키는 로컬 `.env` 또는 배포 환경의 비밀 저장소에만 보관하고 커밋하지 마세요. `.env.example`에는 가짜 예시값만 추가합니다.

## 변경 전 확인

### 코드와 기능

- 새 동작의 정상 경로와 실패 경로를 확인합니다.
- 외부 API가 없어도 오류 메시지가 이해 가능하고 안전하게 실패하는지 확인합니다.
- 서버 전용 키가 브라우저 응답, 로그, 정적 파일에 포함되지 않는지 확인합니다.
- 추천 점수와 설명이 동일한 입력에 대해 재현 가능한지 확인합니다.

### 접근성

JobBridge는 WCAG 2.2 AA 수준을 지향합니다. UI 변경에는 다음을 확인해 주세요.

- 키보드만으로 모든 동작을 실행할 수 있습니다.
- 초점 위치와 초점 표시가 명확합니다.
- 입력 요소에 프로그램적으로 연결된 레이블과 오류 설명이 있습니다.
- 색상만으로 상태를 전달하지 않으며 텍스트 대비를 확인했습니다.
- 동적 상태 변화는 필요한 경우 보조기술에 전달됩니다.
- 확대, 좁은 화면, 동작 감소 설정에서 핵심 기능이 유지됩니다.
- 이미지와 스크린샷에는 의미 있는 대체 텍스트가 있습니다.

접근성 자동 검사만으로 완료로 판단하지 말고 키보드와 대표 보조기술로 수동 확인한 범위를 PR에 적어 주세요.

### 데이터와 모델

데이터 또는 모델 변경에는 다음 내용을 재현 가능한 문서로 남겨 주세요.

- 원출처 URL, 취득일, 버전 또는 스냅샷 식별자
- 원 라이선스와 필요한 저작자 표시
- 전처리 단계와 실행 명령
- 학습/검증 분리 방식, 평가 지표, 알려진 편향과 제한
- 원본 데이터 및 모델 가중치의 공개·재배포 가능 여부

실제 개인정보나 민감정보를 테스트 데이터로 사용하지 마세요. 합성 데이터는 합성임을 명시하고 실제 사람을 재식별할 수 없도록 검토합니다.

## 커밋과 PR

명령형의 짧은 커밋 제목을 권장합니다.

```text
Fix keyboard focus after recommendation load
Document training-data attribution
```

PR 설명에는 문제, 변경 내용, 검증 방법, 접근성·개인정보·라이선스 영향을 포함합니다. UI 변경은 개인정보가 없는 전후 화면을 첨부해도 좋습니다.
